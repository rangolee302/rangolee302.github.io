Project

Survival Camping

multiplayer survival game prototype

About

Survival Camping is a game that, as name suggested, put surviving to camping. It is designed to be a single match co-op game that players would work together to get through their circumstance using the tools they have. It is still in prototype stage which built using Unity.

Idea

One thing that makes players felt tedious is spending time on repeated resource collection, crafting and routines that makes thems survive in game. Survival Camping would make these element critical that player should decide what resource they the most and what should be done first at the moment. The design shift resource collection to resource management, enhancing the feeling of survival.

Multiplayer

The prototype use Photon Unity Netwokring (PUN) 2 as a backbone of netcode. There were lots of choice when choosing the framework for building multiplayer game. But PUN was chosen for easy acesssibility, large community and ease of use.

Multiplayer Cont.

The prototype can create a room that commodate up to 4 players. The host could start the game once every players is ready. Data like nicknames, player's health, object position and state of the game are synchronized and probagate to every client.

The way we synchronize data depends. For objects postiion and animation, I use Photon Transform which done the job itself. And OnPhotonSerializeView used to synchronize simple data like health and game timer. Some complex function we mostly use Remote Procedual Calls to pass a command to host and probagate to others.

Player data storage

Survival Camp will use Firestore on Firebase to store player data. It is planned to have email registration for a account and will further develop a features to save player items, activities and matches result using Firestore. To test that, I made a successful dummy registration using of Firebase Cloud Function and FireStore.

Procedual generate map

The map of each match in Survival Camping will be procedual generated. Procedual generate map has the advantage of randomess that makes each game will not repeat and resource will be allocated randomly. Each game is unqiue is a key to player engagment.
